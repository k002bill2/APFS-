# 농림수산식품모태펀드 투자자산관리시스템 — 대시보드 UI 생성 프롬프트

## 0. 역할과 목표

너는 공공·금융 도메인 엔터프라이즈 대시보드를 설계·구현하는 시니어 프론트엔드 디자인 엔지니어다. "농림수산식품모태펀드 투자자산관리시스템 재구축(1단계)"의 **대시보드 UI/UX를 더미 데이터로 동작하는 정적 React 프로토타입**으로 만든다.

한 줄 요약: 포레스트 그린 톤의 신뢰감 있는 RBAC 대시보드 셸 위에, 모태펀드 투자자산의 KPI·리스크·운용사 건전성·회계마감·투자성과를 One-View로 보여주는 화면 5종을 구현한다.

산출 범위(고정):
- 메인 종합(Executive Overview) 1개
- 서브 대시보드 4개: ① 조기경보 리스크(Risk) ② 운용사 건전성(GP Health) ③ 회계·자금 마감(Accounting & Closing) ④ 투자 성과·포트폴리오(Investment Performance & Compliance)
- 위 5개를 품는 전역 셸(GNB/LNB/브레드크럼/알림센터) + RBAC 게이팅 + 라이트/다크 디자인 시스템

범위 경계(명시):
- 본 5개 대시보드는 PRD 13개 기능영역 중 대시보드 성격이 강한 영역을 선별한 것이다. **FR-5.12 농식품경영체 정보 조회(One-View)는 LNB 메뉴(회계 > 농식품경영체) leaf로만 포함하고, 별도 전용 대시보드 위젯은 현 범위에서 제외**한다(필요 시 후속 확장). 13개 영역 ↔ 화면 매핑은 부록 추적표 참조.

작업 원칙:
- PRD에 근거한 지표/위젯만 만든다(없는 기능 창작 금지). 위젯 배치·시각화 선택은 합리적으로 설계한다.
- 모든 데이터는 한국어 더미. 백엔드 연동 없이 화면이 완결되게 한다.
- 토큰·공통 컴포넌트를 먼저 만들고, 모든 화면은 그 위에서 조합한다(7장 진행 방식 참조).

## 1. 기술 스택 & 프로젝트 규약

스택(고정, 변경 금지):
- React 18 + TypeScript + Tailwind CSS + shadcn/ui + Recharts(차트) + lucide-react(아이콘 전용).
- 아이콘은 lucide-react만 사용. react-icons / heroicons / fontawesome / 이모지 금지. 존재하는 export명만 사용(이 프롬프트가 지정한 아이콘명은 모두 실재 export다).
- 차트는 Recharts만. Recharts로 표현 불가한 4종 특수 위젯(캘린더 마감, 5단계 스텝퍼, 감사 타임라인, 보수정산 산식)은 shadcn/Tailwind 커스텀으로 구현하고, 워터폴/게이지는 각각 BarChart(투명 베이스 스택)/RadialBarChart로 근사한다.

## 2. 디자인 시스템

### 2.1 도메인 톤
농림수산식품 투자자산 시스템이다. 기본색은 숲(forest green). 일반 AI 대시보드의 디폴트 파랑(blue-500)을 쓰지 않는다. 네이비(navy)는 강조/포커스/링크 등 액션 신호에만 제한적으로 쓰는 보조 액센트. 회색-위-회색 저대비 화면 금지.

팔레트(디자인 결정 — PRD는 색 팔레트를 지정하지 않음. 농림수산식품 도메인 정체성에 맞춰 선정한 값이므로 PRD 강제 제약이 아니라 변경 가능한 디자인 토큰이다): Primary `#2C5F2D`(forest) / Secondary `#97BC62`(moss) / Accent `#1E2761`(navy) / Background `#F5F7F4`(cream). 상태색: 정상=green / 주의=amber `#F59E0B` / 경고=red `#DC2626`.

상태색 사용 규칙(고정):
- 정상=success / 주의=warning / 경고=danger (PRD 5.7-01의 3단계와 1:1).
- 증감 색의 의미는 "좋음/나쁨"이지 부호가 아니다. 일반 지표는 양(+)=success, 음(−)=danger. 단 "비용/위반삭감/손실 감소"처럼 음수가 좋은 맥락은 의미 반전 허용.
- 의미를 색에만 의존하지 말 것: 상태/증감/Pass·Fail은 항상 `색 + 아이콘 + 텍스트` 3중 표기.

### 2.3 타이포그래피
- 폰트: Pretendard(`Pretendard Variable` 우선, fallback `-apple-system, "Apple SD Gothic Neo", system-ui, sans-serif`).
- 모든 정량값(금액·비율·IRR·AUM)에 `font-variant-numeric: tabular-nums` 적용(유틸 `.tabular`). 큰 숫자는 `tracking-tight`.
- 금액 단위는 숫자 본체보다 작고 약하게 분리 표기(예: `45.0` + 작은 `억원`). 통화/% 기호도 약하게.
- 스케일: Display(KPI 대형) 36px/1.1·700·tabular / H1 24px/1.3·700 / H2 18px/1.4·600 / CardTitle 15px/1.4·600 / Body 14px/1.6·400 / Label·표헤더 13px/1.4·600·muted / Caption 12px/1.4·400·muted.
- 한글: `word-break: keep-all` 전역, 본문 `leading-relaxed`.

### 2.4 간격 · radius · shadow
- Spacing: 카드 내부 `p-5`(조밀 표 카드 `p-4`), 카드 간 `gap-4`, 섹션 간 `gap-6`, 페이지 `px-6 pt-6`.
- Radius: 카드 `rounded-xl`, 배지/칩/버튼 `rounded-md`, 컬러칩 `rounded-lg`, 점 `rounded-full`.
- Elevation(그림자 절제, 보더 우선): 기본 카드=`border border-border` + `shadow-sm` + `bg-card` / 강조 KPI=`shadow-md`, hover `shadow-lg` / 팝오버·모달=`shadow-lg`. 그림자는 중립 검정 알파 4~8%. 컬러 글로우 금지.

### 2.5 컴포넌트 규약 (공통 래퍼)
모든 위젯은 shadcn `Card` 베이스 + 아래 모티프를 입힌 공통 래퍼로 통일: `StatCard`, `ChartCard`, `DataCard`, `StatusBadge`, `FilterChip`, `Sparkline`.

- `StatCard`: [좌측 4px 컬러바] → (컬러칩 아이콘 + 라벨 muted) → 큰 숫자(Display·tabular·단위작게) → (증감 화살표+값+"전월/목표 대비" 캡션) + 우측 미니 스파크라인(Recharts LineChart, 축·그리드 없음, 높이 ~40px). 목표 대비 지표는 숫자 아래 6px progress(track=muted, fill=달성 success/미달 warning) + "달성/목표" 캡션.
- `StatusBadge`: `inline-flex items-center gap-1.5 rounded-md px-2 py-0.5 text-xs font-semibold` + soft 배경 + 본색 텍스트 + 좌측 색점 + 라벨 텍스트 항상 동반. 매핑: 정상→success, 주의→warning, 경고→danger, Pass→success, Fail→danger, 미마감→muted, 진행중→warning, 완료→success, 휴일→muted(점선).
- `ChartCard` 헤더: 좌측[컬러칩 아이콘 + 제목 + 보조 캡션], 우측[기간 필터(Tabs/Select: 월/분기/연) + 더보기 `MoreHorizontal` ghost]. 헤더 하단 `border-b`. 본문 `pt-4`.
- 테이블(zebra·세로선 금지): 행 구분은 `divide-y divide-border`만. 헤더 `bg-muted/50` + Label 스펙. 행 hover `hover:bg-muted/40`. 숫자 컬럼 `text-right tabular`, 텍스트 `text-left`, 중앙정렬은 배지/아이콘 컬럼만. 행 높이 44px. 정렬 헤더에 `ChevronsUpDown`(14).
- 탭: 활성 트리거 `bg-card shadow-sm text-primary font-semibold`(밑줄 강조선 금지). `FilterChip`: `rounded-md border px-3 py-1 text-sm`, 선택 시 `bg-primary/10 border-primary text-primary`, 해제 `X`(12).

### 2.6 시각 모티프 (필수, 전 카드 일관)
"컬러 액센트 바 + 컬러칩 아이콘" 시그니처를 모든 주요 카드에 적용한다.
1) 카드 좌측 4px 세로 보더(`border-l-4`). 색=카드 의미(일반=primary, 상태=상태색, 회계 자금원천=원천별 고정색 ↓).
2) 카드 헤더 아이콘은 항상 둥근 컬러칩 안에: `inline-flex items-center justify-center size-9 rounded-lg`, 배경=의미색 soft 틴트(`bg-primary/10`, `bg-success-soft` 등), 아이콘=의미색 본색. 맨몸 아이콘 노출 금지(버튼 내부 아이콘만 예외).

자금 원천 4종 고정색(`lib/tokens.ts`의 `FUND_SOURCE_COLORS`, 전 화면 동일): 농식품 모태=chart-1(forest) / 수산 모태=chart-4(teal) / 운영비=chart-5(gold) / 기타 사업=chart-6(brown).

### 2.7 아이콘 규약 (lucide-react)
- strokeWidth: 기본 2, 인라인(배지/증감) 2.5, 큰 장식 1.75. 크기 4단계만: 14(인라인)/16~18(버튼·헤더)/20(컬러칩)/28~32(빈 상태).
- 같은 개념엔 항상 같은 아이콘(`ICON_MAP` 상수). 도메인 매핑: 펀드/자산=`Landmark`·`Wallet`, 리스크/경고=`ShieldAlert`·`AlertTriangle`, 성과/IRR=`TrendingUp`, 운용사/GP=`Building2`, 회계/전표=`FileText`·`Calculator`, 마감/캘린더=`CalendarCheck`·`CalendarDays`, 감사로그=`ScrollText`, 통계=`BarChart3`·`PieChart`, 알림=`Bell`, 잠금/권한=`Lock`.

### 2.8 차트(Recharts) 스타일
- 색은 `var(--chart-1..6)`만(무지개/형광/파랑 금지). 모든 차트 `<ResponsiveContainer>` + 한국어 `<Tooltip>` 포매터(통화=억/백만원, % 소수1자리).
- 그리드 가로선만(`vertical={false}`), `stroke="hsl(var(--border))"` 점선 `3 3`. 축선 제거(`axisLine={false} tickLine={false}`), 틱=caption·muted, 숫자축 tabular.
- 막대 `radius={[6,6,0,0]}`, 라인 `strokeWidth={2}` `type="monotone"`, 면적은 토큰색 그라데이션(상단 0.25→하단 0.02). 진입 애니메이션 `animationDuration={400}` 1회만(반복/바운스 금지).
- 임계선은 `ReferenceLine`(danger 점선). 벤치마크 비교는 자펀드=primary 실선, BM=muted 점선.

## 6. UX 가드레일 (전 화면 무조건 적용 — 위반 시 재생성)

- 명도대비 4.5:1 이상(상태색 위 텍스트·cream 위 moss 등 저대비 조합 보정, 다크 동일 기준).
- 의미를 색에만 의존 금지: 상태/증감/Pass·Fail은 `색 + lucide 아이콘 + 텍스트` 3중.
- 키보드만으로 전 기능(Tab 논리적 순서, Enter/Space 실행, Esc 모달 닫기). `tabIndex` 남용 금지.
- 포커스 링 항상 가시(`focus-visible:ring-2 ring-ring ring-offset-2`), `outline:none` 단독 금지, shadcn 포커스 제거 금지.
- 아이콘 전용 버튼 `aria-label`(한국어), 장식 아이콘 `aria-hidden`. 동적 알림 `aria-live="polite"`.
- 차트 접근성: 차트마다 제목/요약 `aria-label` + "데이터 테이블로 보기" 토글로 접근 가능한 `<table>` 제공(차트 단독 정보전달 금지).
- 시맨틱 `<header>/<nav>/<main>/<section>` + WAI-ARIA, 제목 레벨(h1→h2→h3) 건너뛰기 금지.

반응형·브라우저:
- 브레이크포인트 3종: Mobile<768 / Tablet 768–1279(`md`) / PC≥1280(`xl`). 그리드 12컬럼(PC 카드 4·차트 2/행, Tablet 6, Mobile 1).
- 모바일 축약: 대용량 표→카드형/핵심컬럼+행 펼침, 멀티 차트 범례 하단·라벨 축약, 필터바→"필터" 시트.
- Edge/Chrome/Safari/Whale 동일 렌더(표준 CSS만, 벤더 종속·실험 API 금지), 가로 스크롤 깨짐 금지, 터치 타깃 ≥44×44px.

상태 처리: 5.x 공통 5상태(로딩/빈/필터0건/에러/권한없음) 동일 패턴. 에러 메시지에 스택·SQL·서버 경로 등 내부정보 노출 금지(NFR-SEC-023).

숫자/날짜 포맷:
- 천단위 콤마 필수, 금액=원/억원(예: 30억원, 4,500,000,000원), 비율=% 소수1자리(IRR +32.5%, 80.0%).
- 수치 셀 우측 정렬, 음수=danger+부호(−5,000,000), 양수 증감=success+`+`(색 단독 금지).
- 전월/전기 대비 변동 항목 하이라이팅(FR-5.2-06). 주민번호 등 PII 마스킹(`******`, NFR-SEC-010 / FR-5.13-04).
- 날짜 `YYYY-MM-DD`, 일시 `YYYY-MM-DD HH:mm:ss`(초 단위). 감사로그·이력은 초 단위 필수(예: 2026-01-21 14:32:05, FR-5.11-03 / NFR-SEC-014).

엑셀 다운/업로드: 표/위젯 우상단 액션 영역에 `Download`/`Upload` + "엑셀 다운로드"/"엑셀 업로드" 일관 배치. 업로드는 양식 다운로드 → 업로드 → 실패 행/사유 결과 표시 흐름 UI. 등록원부 등 PDF 발급 화면은 "PDF 다운로드" 병행. (FR-5.1-08, 5.3-03, 5.8-08/09, 5.11-07)

내비게이션: 전 페이지 GNB+LNB+브레드크럼(활성 메뉴 강조+`aria-current`), 사이트맵 경로. KPI/카드 → 상세·업무 화면 드릴다운(4.8). 메뉴는 RBAC 동적 노출(권한 외 미렌더).

## 8. 피해야 할 것 (Anti-patterns)
1. 디폴트 파랑(blue-500류) 기본값 금지 — 기본 forest, 액센트만 navy.
2. raw hex / 일반 Tailwind 팔레트 클래스 직접 사용 금지(토큰 경유).
3. 무지개 차트 팔레트·형광색·이모지·타 아이콘셋 금지.
4. 제목/헤더 밑줄 강조선 금지 — 강조는 굵기·색·컬러칩으로.
5. 모든 카드 동일 크기 무한 나열 금지 — KPI 행/비대칭 차트 그리드/풀폭 표로 위계.
6. 본문·수치 중앙정렬 금지(라벨 좌·숫자 우 기본, 중앙은 배지/아이콘/빈상태만).
7. 저대비 회색-위-회색 금지 — muted-foreground는 캡션/메타 전용, 본문 사용 금지.
8. zebra 테이블·세로 구분선 금지(가로 divide만).
9. 그림자 글로우·컬러 섀도·과한 elevation 금지(보더 우선).
10. 의미를 색에만 의존 금지(항상 아이콘+텍스트 병기).
11. PRD에 없는 지표/기능 창작 금지(지도 채색 등 미지원은 대안 명시 + 확장 슬롯).
12. 반복/바운스 차트 애니메이션 금지(데이터 신뢰감 우선).
