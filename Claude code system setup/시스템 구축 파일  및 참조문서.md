10. **Workflow 도구 (인라인 JS 오케스트레이션)**: 스크립트 상단에 `export const meta = { name, description, phases }` (순수 리터럴) 선언으로 시작. 본문에서 전역 함수를 직접 `await`로 사용 — `agent(prompt, opts?)`(서브에이전트; `opts = { label, phase, schema, model, effort, isolation:'worktree', agentType }`, `schema` 지정 시 검증된 객체 반환), `parallel(thunks)`(동시 실행 + 배리어, 실패 항목은 `null`), `pipeline(items, stage1, stage2, ...)`(항목별 독립 파이프라인, 단계 간 배리어 없음), `phase(title)`, `log(message)`, `args()`, `budget()`(토큰 예산), `workflow(name, args)`(중첩 1단계). 동시 실행은 `min(16, cpu코어-2)`로 자동 캡, 총 1000 에이전트 상한. 백그라운드 실행 후 완료 알림.
11. **advisor (검증 게이트)**: 전체 트랜스크립트를 보는 상위 리뷰어(파라미터 없음). 접근법 확정 전 / 완료 선언 전에 호출하는 검증 게이트.
12. **Agent Teams / Subagents (최신)**: `Agent` 도구로 서브에이전트 실행 — 옵션 `subagent_type`, `run_in_background`, `isolation:'worktree'`(자체 git worktree, 미변경 시 자동 정리). `SendMessage`로 기존 에이전트(ID/이름)에 컨텍스트 유지한 채 이어서 지시. 빌트인 타입: `general-purpose`, `Explore`(읽기전용 탐색), `Plan`, `planner`, `architect`, `code-reviewer`, `security-reviewer` 등. 플러그인 스코프 에이전트는 `ecc:*` 네임스페이스로 공존(예: `ecc:architect`, `ecc:python-reviewer`) — "플랫 빌트인 + 플러그인 스코프(`ecc:*`) 공존" 구조. [Agent Teams 공식 문서](https://code.claude.com/docs/ko/agent-teams)
13. **MCP 생태계**: claude-in-chrome, Figma, Canva, Atlassian, Notion, Slack, Gmail, Google(Calendar/Drive), Context7, Playwright 등 다수. `ToolSearch`로 디퍼드(지연 로드) 도구 스키마를 동적 로드.
