## 🎯 실제 워크플로우 예제

### 예제 1: 새 React 프로젝트 시작하기

#### Step 1: 프로젝트 초기화 (macOS)
```bash
# 프로젝트 생성 (Next.js 15 최신)
npx create-next-app@latest my-app --typescript --tailwind --app
cd my-app

# Claude Code 설정
mkdir -p .claude/skills .claude/commands .claude/agents .claude/plugins
```

#### Step 2: CLAUDE.md 작성
```markdown
# My App - E-commerce Platform

## Project Overview
- **Purpose**: Online shopping platform
- **Tech Stack**: Next.js 15, TypeScript, Tailwind CSS, Drizzle ORM
- **Features**: Product catalog, Cart, Checkout, User auth

## Current Tasks
1. Set up authentication with Supabase Auth
2. Create product listing page
3. Implement shopping cart
4. Add payment integration

## Code Standards
- Use Server Components by default
- Client Components only when needed
- Proper error boundaries
- Loading states for all async operations
```

#### Step 3: Claude Code 세션 시작
```bash
claude

# effort 설정
/effort high

# 첫 작업 요청
"Using the project context in CLAUDE.md, set up the basic authentication system with Supabase Auth and create the initial user model"

# Sub-agent 생성
/agents
# Create 'auth-specialist' agent for authentication tasks

# Skill 생성
"Use the skill-creator to create a skill for generating Drizzle ORM models"
```

### 예제 10: Workflow 도구로 다단계 자동화 (신규)

#### Workflow 스크립트로 "계획 → 병렬 구현 → 항목별 파이프라인" 오케스트레이션

> Workflow 도구는 **인라인 JS 스크립트**다. 반드시 순수 리터럴인
> `export const meta = { name, description, phases }` 로 시작하고,
> 본문에서 전역 함수(`agent`, `parallel`, `pipeline`, `phase`, `log`, `args`)를
> `await` 로 직접 사용한다. (※ 2026-06-16 갱신: `workflow.pipeline([...])`,
> `.withIsolation().run()`, `workflow.background()` 같은 fluent 체인은 실재하지 않는다.)

```js
// 사용자 대시보드 기능을 계획 → 병렬 구현 → 검증까지 자동화하는 Workflow
export const meta = {
  name: "dashboard-feature",
  description: "대시보드 기능을 계획하고 백엔드/프론트를 병렬 구현 후 검증",
  phases: ["plan", "build", "verify"],
};

// 1) 계획 단계 — schema를 주면 검증된 객체를 그대로 반환받는다
phase("plan");
const plan = await agent(
  "사용자 대시보드(주문 내역/즐겨찾기/계정 설정)의 구현 계획을 세워라. " +
    "백엔드 API와 프론트엔드 컴포넌트 작업을 분리해 작업 목록으로 나눠라.",
  {
    label: "planner",
    phase: "plan",
    agentType: "Plan",
    effort: "high",
    schema: {
      type: "object",
      properties: {
        backendTasks: { type: "array", items: { type: "string" } },
        frontendTasks: { type: "array", items: { type: "string" } },
      },
      required: ["backendTasks", "frontendTasks"],
    },
  },
);
log(`백엔드 ${plan.backendTasks.length}건 / 프론트 ${plan.frontendTasks.length}건`);

// 2) 구현 단계 — parallel은 "함수(thunk) 배열"을 받아 동시 실행 + 배리어(모두 완료 대기)
//    각 에이전트는 자체 git worktree에서 격리 실행된다 (미변경 시 자동 정리)
phase("build");
const [backend, frontend] = await parallel([
  () =>
    agent(`다음 백엔드 작업을 구현하라:\n${plan.backendTasks.join("\n")}`, {
      label: "backend",
      phase: "build",
      isolation: "worktree",
      agentType: "general-purpose",
    }),
  () =>
    agent(`다음 프론트엔드 작업을 구현하라:\n${plan.frontendTasks.join("\n")}`, {
      label: "frontend",
      phase: "build",
      isolation: "worktree",
      agentType: "general-purpose",
    }),
]);
// 실패한 항목은 null로 들어온다
if (!backend || !frontend) log("일부 구현 실패 — 검증 단계에서 확인 필요");

// 3) 검증 단계 — pipeline은 항목별 독립 파이프라인(단계 간 배리어 없음)
//    각 모듈을 "리뷰 → 테스트"로 흘려보낸다
phase("verify");
await pipeline(
  ["backend", "frontend"],
  (mod) =>
    agent(`${mod} 모듈을 code-reviewer 관점으로 검토하라`, {
      phase: "verify",
      agentType: "code-reviewer",
    }),
  (mod) =>
    agent(`${mod} 모듈의 build/test/lint를 실행하고 결과를 보고하라`, {
      phase: "verify",
    }),
);
```

> Workflow는 동시 실행을 자동으로 `min(16, CPU 코어-2)`로 캡하고,
> 총 1000 에이전트 상한을 둔다. 백그라운드로 실행되며 완료 시 알림이 온다.

## 💡 고급 팁과 트릭

### 1. Multi-Agent 협업
```bash
# 복잡한 기능을 여러 agent가 협업
"I need to implement a real-time notification system.
Coordinate between:
- Plan agent for system design
- architect for architecture
- general-purpose for implementation
- test-automation-specialist for testing"
```

### 2. Context Preservation
```bash
# 긴 작업을 여러 세션으로 분할
/compact  # 컨텍스트 압축
"Save progress to dev docs"

# 또는 메모리로 핵심 정보 보존
"Remember: We're at Phase 2, Step 3 of the dashboard implementation"
```

### 3. Automated Workflows
```json
// .claude/settings.json - 자동 워크플로우
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write(*.tsx)",
        "hooks": [
          {"type": "command", "command": "npx tsc --noEmit"},
          {"type": "command", "command": "npm run lint:fix"}
        ]
      }
    ],
    "PreCompact": [
      {
        "hooks": [{
          "type": "command",
          "command": "echo 'Context will be compacted - check dev docs'"
        }]
      }
    ]
  }
}
```

### 4. Performance Optimization
```bash
# 큰 파일 처리 시
"Process src/data/large-file.json in chunks of 1000 records"

# 간단한 작업은 빠르게
/fast
"Update all import paths from @old to @new"
```

## 🚨 일반적인 문제 해결

### Case 1: Skill이 동작하지 않을 때
```bash
# 디버깅 단계
1. "List all available skills"
2. SKILL.md의 description 필드 구체화
3. YAML frontmatter 형식 확인
4. /compact 후 재시도
```

### Case 2: Agent가 잘못된 작업 수행
```bash
# Agent 수정
/agents
# Edit agent description to be more specific
# Limit tool access with disallowedTools
# Set maxTurns to prevent runaway execution
# Add constraints in system prompt
```

### Case 3: 컨텍스트 오버플로우
```bash
# 해결 방법
1. /compact로 컨텍스트 압축
2. 작업을 작은 단위로 분할
3. 불필요한 파일 컨텍스트 제거
4. Dev Docs에 진행 상황 저장 후 /clear
```

### Case 4: 메모리 관련 문제
```bash
# 메모리 확인
/memory

# 메모리가 로드되지 않으면
1. MEMORY.md 인덱스 파일 확인
2. 메모리 파일 frontmatter 형식 확인
3. description 필드가 구체적인지 확인
```
