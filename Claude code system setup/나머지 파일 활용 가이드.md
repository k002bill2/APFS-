# 나머지 파일 활용 가이드

> 최종 업데이트: 2026-06-16
> Claude Code: v2.1.143 | Models: Opus 4.8 / Fable 5 / Sonnet 4.6 / Haiku 4.5

## 개요

[[claude_code_setup_prompt]]가 자동으로 설치하는 3대 시스템 외에, 이 폴더에는 **템플릿형**과 **참조형** 파일이 있습니다. 이들은 자동 설치 대상이 아니라 **수동으로 참고하거나 복사하여 사용**합니다.

## 템플릿형 파일 사용법

### 1. CLAUDE.md 템플릿

**파일**: [[CLAUDE.md 템플릿]]
**용도**: setup prompt가 생성한 기본 CLAUDE.md를 프로젝트에 맞게 보강할 때 사용
**사용법**:

```bash
# setup prompt 실행 후, 생성된 CLAUDE.md에 템플릿 내용을 병합
# Claude Code에서:
> "CLAUDE.md를 확장해줘.
>  이 템플릿을 참고해: skills guide/CLAUDE.md 템플릿.md
>  Code Style, Git Workflow 섹션을 추가해"
```

> [!tip] 핵심
> setup prompt는 에이전트/안전 원칙 중심의 CLAUDE.md를 생성합니다.
> 이 템플릿은 **코딩 규칙, 테스트 전략** 등 개발 가이드라인을 추가합니다.
> 둘을 합쳐야 완전한 CLAUDE.md가 됩니다.

---

### 2. 프로젝트별 템플릿

**파일**: [[프로젝트별 템플릿]]
**용도**: 프로젝트 타입에 따라 최적화된 설정을 복사
**포함된 템플릿**:

| 프로젝트 타입 | CLAUDE.md | Skills | settings.json |
|-------------|:---------:|:------:|:-------------:|
| React/Next.js | O | O | O |
| Python/FastAPI | O | O | O |
| Node.js Backend | O | O | O |
| ML/Data Science | O | O | O |
| Mobile (React Native) | O | O | O |

**사용법**:

```bash
# 해당 프로젝트 타입의 섹션을 찾아서 관련 부분을 복사
# 예: React 프로젝트일 경우

> "프로젝트별 템플릿.md에서 웹 개발 프로젝트(React/Next.js) 섹션을 읽고,
>  현재 CLAUDE.md에 Tech Stack과 Development Guidelines를 병합해줘.
>  Skills 구성도 .claude/skills/에 추가해줘."
```

---

### 3. Agent Skills 예시 모음

**파일**: [[Agent Skills 예시 모음]]
**용도**: 검증된 스킬 코드를 .claude/skills/에 복사하여 사용
**포함된 스킬 예시**:

- Code Reviewer
- Frontend Dev Guidelines
- Backend Dev Guidelines
- Database Verification
- Performance Optimizer
- 등

**사용법**:

```bash
# 방법 1: Claude에게 직접 요청
> "Agent Skills 예시 모음.md를 읽고,
>  code-reviewer 스킬을 .claude/skills/code-reviewer/SKILL.md로 생성해줘"

# 방법 2: 수동 복사
mkdir -p .claude/skills/code-reviewer
# 예시 모음에서 해당 코드 블록을 복사하여 SKILL.md에 붙여넣기
```

> [!warning] skill-rules.json 업데이트 필수
> 새 스킬을 추가한 후, `.claude/hooks/skill-rules.json`에도 해당 스킬의 트리거 규칙을 추가해야 자동 활성화가 동작합니다.

## ※ 2026-06-16 갱신: 모던 Claude Code 도구/스킬 참조

이 폴더의 참조 문서들이 작성된 이후 추가된 핵심 도구/스킬입니다. 위 참조형 파일을 읽을 때 아래 항목을 함께 염두에 두세요.

### 모델 라인업

| 모델 | ID | 비고 |
|------|----|------|
| Opus 4.8 | `claude-opus-4-8` | 플래그십, 1M 컨텍스트 변형 존재 |
| Fable 5 | `claude-fable-5` | 신규 세대 |
| Sonnet 4.6 | `claude-sonnet-4-6` | 1M 컨텍스트 |
| Haiku 4.5 | `claude-haiku-4-5-20251001` | 200K 컨텍스트 |

> [!note] `/fast` 모드
> `/fast`는 작은 모델로 다운그레이드하는 것이 아니라 Opus(4.8/4.7/4.6)를 빠른 출력 모드로 실행합니다.

### Workflow 도구 (인라인 JS 오케스트레이션)

**언제 쓰나**: 여러 서브에이전트를 코드로 조율하는 파이프라인을 구성할 때. (위 [[CLI  Orchstration 설정]]의 코드 기반 대안)

- 반드시 `export const meta = { name, description, phases }` (순수 리터럴)로 시작합니다.
- 본문 전역 함수(직접 `await` 사용):
  - `agent(prompt, opts?)` — 서브에이전트 실행. `opts = { label, phase, schema, model, effort, isolation: 'worktree', agentType }`. `schema`를 주면 검증된 객체를 반환.
  - `parallel(thunks)` — 동시 실행 + 배리어(모두 완료 후 반환). 실패 항목은 `null`.
  - `pipeline(items, stage1, stage2, ...)` — 항목별 독립 파이프라인, 단계 간 배리어 없음.
  - `phase(title)`, `log(message)`, `args(입력값)`, `budget(토큰 예산)`, `workflow(name, args)`(중첩 1단계).
- 동시 실행은 `min(16, CPU코어-2)`로 자동 캡, 총 1000 에이전트 상한. 백그라운드 실행 후 완료 시 알림.

> [!warning] 날조 API 주의
> `workflow.pipeline([{task, agent, depends}])`, `.withIsolation().run()`, `workflow.background()` 같은 fluent builder 체인은 **존재하지 않습니다**. 위의 전역 함수 시그니처만 사용하세요.

### 서브에이전트 (Agent 도구)

- `Agent` 도구로 서브에이전트 실행. 옵션: `subagent_type`, `run_in_background`, `isolation: 'worktree'`(자체 git worktree, 미변경 시 자동 정리).
- `SendMessage`로 기존 에이전트(ID/이름)에 컨텍스트를 유지한 채 이어서 지시.
- 빌트인 타입: `general-purpose`, `Explore`(읽기전용 탐색), `Plan`, `planner`, `architect`, `code-reviewer`, `security-reviewer` 등.
- 플러그인 스코프 에이전트는 `ecc:*` 네임스페이스로 공존(예: `ecc:architect`, `ecc:python-reviewer`). 즉 **플랫 빌트인 + 플러그인 스코프(ecc:*) 공존** 구조입니다(완전 평탄화 아님).

### 신규 도구/스킬

- **advisor** — 전체 트랜스크립트를 보는 상위 리뷰어(파라미터 없음). 접근법 확정 전 / 완료 선언 전에 호출하는 검증 게이트.
- **ScheduleWakeup** — `/loop` 동적 페이싱 재개 스케줄.
- **CronCreate / CronList / CronDelete** — cron 예약 에이전트.
- **EnterWorktree / ExitWorktree** — worktree 라이프사이클. **Monitor** — 백그라운드 프로세스/조건 모니터링.
- **Skill** 도구 — 스킬 명시 호출. **ToolSearch** — 디퍼드(지연 로드) 도구 스키마 동적 로드.
- **SendUserFile** — 산출 파일을 사용자에게 전달. **PushNotification / RemoteTrigger / DesignSync**.
- 빌트인 스킬: `deep-research`, `skill-creator`, `frontend-design`, `simplify`, `document-skills`(xlsx/docx/pptx/pdf), `/code-review ultra`(=ultrareview; 멀티에이전트 클라우드 리뷰, 사용자 트리거).
- 반복/예약: `/loop` 스킬(반복 실행), `/schedule` 스킬(cron 클라우드 에이전트, routine).
- MCP 생태계: claude-in-chrome, Figma, Canva, Atlassian, Notion, Slack, Gmail, Google(Calendar/Drive), Context7, Playwright 등 다수(`ToolSearch`로 동적 로드).
