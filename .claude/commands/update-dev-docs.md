description: Update dev docs before context compaction or session end
---

Find the active dev docs in `dev/active/` and update them:

1. **context.md**
   - Update the "Last Updated" timestamp
   - Add any new decisions made this session
   - Update the "Current Issues" section
   - Revise "Next Steps" based on progress

2. **tasks.md**
   - Mark completed items with `[x]`
   - Add any new tasks discovered
   - Update completion counts
   - Reorder by priority if needed

3. **Append a session summary** to context.md
   - What was accomplished
   - Any blockers encountered
   - Critical next actions

Keep updates concise but comprehensive. If multiple active tasks exist, update the one currently being worked on (ask if ambiguous).
